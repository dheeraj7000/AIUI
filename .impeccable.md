# AIUI Design Context

## Users

### Primary Audience

Broadly targeted at software engineers and designers ("everyone" for now) who use AI assistants to build UIs.

### Context of Use

Users interact with AIUI in two primary modes:

1. **Bootstrap/Setup**: Configuring a new project's design system from scratch.
2. **Integration/Migration**: Bringing an existing design system into AIUI to control AI output, or exporting from AIUI to a new project.

### Job to be Done

"Give me absolute control over how AI builds my UI so I don't have to manually fix 'AI-slop' design choices every time."

## Brand Personality

### 3-Word Personality

**Clinical / Architectural / Precise**

### Voice & Tone

High-signal, professional, and authoritative. It should feel like a precision tool for engineers, avoiding the "bubbly/playful" SaaS tropes.

### Emotional Goal

**Confidence & Authority**. The user should feel that the design system is a "source of truth" that the AI is strictly bound to follow.

## Aesthetic Direction

### Visual Tone: "Steel & Ink"

A high-contrast, editorial-inspired aesthetic.

- **Palette**: Paper (white/off-white), Ink (deep midnight), Oxblood/Steel Blue (accents).
- **Typography**: Geometric sans for UI (Switzer), refined serif for display (Gambarino), and precise monospace for data/code (Fragment Mono).
- **Form**: Sharp corners (`radius: 0`), thin hairlines (`1px`), and generous, intentional whitespace.

### Design Principles

1. **Signal over Noise**: Remove all decorative elements that don't convey meaning (no glassmorphism, no generic gradients).
2. **Architectural Rigor**: Use a strict 4px grid. Layouts should feel like blueprints or high-end specimen sheets.
3. **Clinical Precision**: Colors and typography must be used to create clear functional hierarchies.
4. **Transition via Mapping**: Maintain a bridge between legacy utilities and the new Editorial system until the full rewrite is complete.
