# AIUI — Design Context

This file is the source of truth for design decisions across AIUI's marketing site and product UI. Read it before any `/impeccable`, `/critique`, `/polish`, `/typeset`, `/colorize`, or related skill runs.

## Users

**Primary user:** anyone building a web application or software product who wants their UI to stay consistent and feel distinctive — not generic.

They come to AIUI because they are already using Claude (or similar) to generate UI code and are tired of:

- Their design drifting every time they generate new UI
- Every AI-generated screen looking like the same purple-gradient SaaS template
- Losing control of the aesthetic decisions they care about

**Context of use:** at a screen, mid-build, with their editor open next to AIUI. They are working, not browsing. They care more about control and predictability than novelty.

**Job to be done:** "I want my design to stay consistent across every screen my AI builds for me."

**Emotion after first use:** _control · consistency · personalization._ The anti-emotion is "the AI took over." AIUI puts the human back in charge.

## Brand Personality

Three words: **calm · exacting · plainspoken.**

- **Calm** — no hype, no neon, no animated gradients screaming for attention. The product is for people doing serious work.
- **Exacting** — tokens, rules, measurements, named sizes. Precision is the product.
- **Plainspoken** — copy is simple, direct English. Never "unleash AI-powered design intelligence." Say what it does.

The voice of a well-edited technical manual, not a growth-hacked landing page.

## Aesthetic Direction

**Concept: editorial design-manual / typography specimen.**

The whole product should feel like a beautifully printed book _about_ design — because AIUI's promise is that _your_ design stays intact. Treating the product surface itself as a design specimen reinforces the premise at every scroll.

Signals of this direction:

- Left-aligned text, never centered body copy
- Horizontal rules as structural devices between sections
- Index numbers / section numerals in the margin (01, 02, 03…)
- Figure + caption pairings instead of identical cards
- Specimen-sheet treatments for tokens, type, and components (show the thing, label it)
- Generous vertical rhythm; asymmetric columns; breakable grid
- One saturated ink color used sparingly — the rarity is the point

**Anti-references (things AIUI must NOT look like):**

- Vercel / Linear / shadcn dark-mode template with glowing purple-blue gradients
- Cyan-on-dark, neon accents, aurora mesh backgrounds, backdrop-blur on everything
- Hero-metric dashboard template (big number + small label + gradient icon box)
- Generic icon-heading-text card grids
- Any landing page that could be for any AI SaaS product

**Theme: light.** Warm paper, dark ink. Rationale: (a) designers evaluate color and contrast against light surfaces; (b) the anti-AI positioning is undermined by the default "dark mode + glow" aesthetic; (c) a well-executed light theme is harder and rarer — it reinforces the craft claim.

**Color system (OKLCH, all neutrals tinted toward the accent hue for cohesion):**

| Token           | Value                   | Use                                                        |
| --------------- | ----------------------- | ---------------------------------------------------------- |
| `--paper`       | `oklch(97.5% 0.008 60)` | Page background — warm off-white, never #fff               |
| `--paper-deep`  | `oklch(94% 0.01 60)`    | Secondary surface (rare; prefer paper + rule)              |
| `--ink`         | `oklch(22% 0.02 30)`    | Primary text — near-black with a warm tint, never #000     |
| `--ink-muted`   | `oklch(45% 0.015 50)`   | Secondary text, captions, marginalia                       |
| `--rule`        | `oklch(85% 0.008 60)`   | Hairlines, dividers, table cell borders                    |
| `--accent`      | `oklch(44% 0.14 28)`    | Oxblood — single saturated ink. Links, key emphasis, marks |
| `--accent-soft` | `oklch(92% 0.04 28)`    | Background wash for accent regions — rare                  |

Usage discipline:

- 60% paper, 30% ink (various weights), 10% accent. Accent works because it's rare — do not tint every CTA.
- No pure black, no pure white, anywhere, ever.
- No gradients on text. Ever. (See impeccable's absolute ban.)
- No border-left/right stripes. (Also banned.)

## Typography

Fonts selected to feel editorial and specific — _not_ the reflex AI picks (Inter, Geist, DM Sans, Plus Jakarta, Space Grotesk, Fraunces, IBM Plex, Instrument are all rejected).

| Role    | Family            | Source              | Use                                                                                       |
| ------- | ----------------- | ------------------- | ----------------------------------------------------------------------------------------- |
| Display | **Gambarino**     | Fontshare (free)    | Marketing headlines, editorial titles, section numerals                                   |
| Body    | **Switzer**       | Fontshare (free)    | All UI copy, body text, labels, form fields                                               |
| Mono    | **Fragment Mono** | Google Fonts (free) | Code blocks, token values, keyboard shortcuts — typewriter character, not programmer-mono |

- **Marketing / content pages:** fluid type with `clamp()` on headings, serif Gambarino for titles, Switzer for body.
- **Product UI / dashboard:** fixed `rem` scale (no fluid type), Switzer only. Gambarino is reserved for marketing surfaces to preserve its specialness.
- Type scale ratio: **1.333** (perfect fourth). Five steps: 0.875 / 1 / 1.333 / 1.777 / 2.369 / 3.157rem.
- Line length capped at 65–72ch on body text.
- Line height: 1.5 on body, 1.1 on display; add +0.05 if a surface ever inverts to light-on-dark.
- Numerals: tabular-nums in tables and token listings; oldstyle figures in body prose where supported.

## Design Principles

1. **Specimen over skeuomorph.** Show the thing, label it, measure it. The UI teaches what AIUI does by being a well-labeled specimen of it.
2. **Rule over shadow.** Use hairline rules (1px `--rule`) to structure the page, not drop shadows on floating cards. Cards are the exception; the rule (literally) is the default.
3. **Rare accent.** Oxblood appears sparingly — for links, the single primary action on a page, key section marks, and token swatches. If it's on more than ~10% of the viewport, remove some.
4. **Plainspoken copy.** Every label, heading, empty state, and error message uses simple direct English. No "unleash," "supercharge," "AI-powered," "intelligent," "seamless." Say what the button does. Say what went wrong.
5. **Craft is in the details.** Generous margins, intentional vertical rhythm, real optical kerning on display type, tabular numerals in tables, `font-feature-settings` enabled. The whole point is to feel designed — not generated.

## Accessibility

- **Target: WCAG 2.2 AA.** AAA contrast where it costs us nothing (body text on paper already exceeds AAA with this palette).
- Every interactive element has a visible focus ring using `--accent` at reduced chroma + `outline-offset: 2px`. Never `outline: none` without replacement.
- Color is never the only signal: status is always accompanied by an icon or text label.
- `prefers-reduced-motion: reduce` removes non-essential transitions; keep only state-change feedback.
- Touch targets ≥ 44×44 on primary actions, ≥ 40×40 on tertiary. Icon-only buttons always carry `aria-label`.
- All toggle / active / selected states use the correct ARIA: `aria-current`, `aria-pressed`, `aria-selected`.

## Scope

This brief covers the full AIUI surface area:

- Marketing / landing (`/`, `/docs`, auth)
- Product UI (`/dashboard`, `/projects`, `/studio`, `/style-packs`, `/components`, `/api-keys`, `/profile`, `/audit-log`, `/import`)

Marketing pages get the full editorial treatment (Gambarino display, fluid type, asymmetric layouts, numbered sections). Product UI uses the same palette and body type but restrains display usage to preserve its weight for marketing contexts.
